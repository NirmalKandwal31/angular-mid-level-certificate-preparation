Hoisting, var vs let

Learn hoisting, scope differences, and why loops behave differently with var and let.

What is Hoisting?

Hoisting is JavaScript behavior where variable and function declarations are processed before the code runs.

  • var is hoisted and initialized with undefined.
  • let is hoisted but cannot be accessed before initialization (this is called the Temporal Dead Zone).
1) Block Scope (var vs let)

var is function-scoped, so it can be accessed outside an if block. let is block-scoped, so it exists only inside the block.

With let, each iteration gets its own block-scoped variable, so callbacks preserve the correct value.

Key Takeaways
  • var is function-scoped and can cause unexpected bugs.
  • let is block-scoped and safer for modern JavaScript.
  • Prefer let and const in Angular apps.
