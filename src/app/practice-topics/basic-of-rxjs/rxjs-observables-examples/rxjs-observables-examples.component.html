Filter

Keep what matches

Definition

filter keeps only those items that satisfy a condition.

When to use

  • Need to remove unwanted items (age > 24, active users, etc.)
  • Apply conditions before mapping or API calls

Benefits

  • Cleaner data pipeline
  • Reduces unnecessary work downstream

Output: users whose age is less than 24.

Map

Transform

Definition

map transforms each item into a new value/object.

When to use

  • Need to change shape of data (pick fields, create new object)
  • Formatting before UI display

Benefits

  • Readable transformations
  • Keeps original data unchanged (immutable)

Output: only the name of each user.

Reduce

Accumulate

Definition

reduce combines all items into one single result (sum, min, max, object, etc.).

When to use

  • Total/summary calculations (total age, total price)
  • Finding min/max item

Benefits

  • One pass calculation
  • Great for summary stats

SwitchMap

Cancel previous

Definition

switchMap maps to an inner observable and automatically cancels the previous inner observable when a new value arrives.

When to use

  • Search box / typeahead (latest request only)
  • Route param change → fetch latest data

Benefits

  • Prevents outdated responses
  • Saves network & avoids race conditions

Output: latest emitted user detail (demo).

ForkJoin

Wait all complete

Definition

forkJoin runs multiple observables in parallel and emits once when all observables complete (last values).

When to use

  • Multiple HTTP calls and need all results together
  • Page load: users + config + permissions etc.

Benefits

  • Parallel execution = faster
  • One combined response, clean code
Don't use: with streams that never complete (events, intervals, form valueChanges), because then it will never emit.
