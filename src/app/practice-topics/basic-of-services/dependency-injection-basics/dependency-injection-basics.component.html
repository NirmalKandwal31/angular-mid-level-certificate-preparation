What is Dependency Injection (DI)?

Injectable Services

Injectable tells Angular that a class can be created and injected by the DI system. Using providedIn: 'root' is the most common approach.

Injecting a Service (Constructor Injection)

Angular injects dependencies through the constructor. You don’t create the dependency with new. Angular creates it and passes it in.

Providers

A provider tells Angular how to create a dependency and in which injector scope it should live.

Injector (Optional)

Angular also provides the Injector API to resolve dependencies manually. Constructor injection is preferred in most cases.

Key Takeaways
  • DI means dependencies are provided to classes instead of created inside them.
  • Injectable enables Angular to inject a service.
  • Providers define how and where an instance is created.
  • Hierarchical injectors decide which instance you receive.
